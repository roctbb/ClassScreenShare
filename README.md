# 📹 Система прокторинга экзаменов

Веб-система для мониторинга экранов участников экзамена в реальном времени с возможностью записи и конвертации в видео.

## ✨ Возможности

- 🖥️ **Просмотр экранов в реальном времени** - отображение всех подключенных участников
- 💾 **Автоматическая запись** - сохранение скриншотов с временными метками
- 🎬 **Конвертация в видео** - преобразование скриншотов в MP4 через ffmpeg
- ⚡ **Оптимизация производительности** - настройка качества и частоты захвата
- 🔔 **Детекция неактивности** - предупреждения о проблемах с подключением
- 🐳 **Docker-ready** - простой запуск через контейнеры
- 🎛️ **Веб-интерфейс управления** - просмотр и управление записями

## 🚀 Быстрый старт

### Вариант 1: Docker (рекомендуется)

```bash
# 1. Клонируйте репозиторий
git clone <url>
cd ClassScreenShare

# 2. Создайте .env файл (или используйте существующий)
cp .env.example .env

# 3. Запустите через Docker
docker-compose up -d

# 4. Откройте в браузере
# Участники: http://localhost:3000
# Преподаватель: http://localhost:3000/screens?password=inga
# Управление: http://localhost:3000/manage?password=inga
```

### Вариант 2: Локальный запуск

```bash
# 1. Установите зависимости
npm install

# 2. Установите ffmpeg (для конвертации видео)
# macOS:
brew install ffmpeg

# Ubuntu/Debian:
sudo apt install ffmpeg

# Windows:
# Скачайте с https://ffmpeg.org/download.html

# 3. Настройте .env файл
nano .env

# 4. Запустите сервер
node share_server.js
```

## ⚙️ Настройки (.env)

```bash
# Основные настройки
PASSWORD=inga                 # Пароль доступа для преподавателя
PORT=3000                     # Порт сервера

# Запись
ENABLE_RECORDING=true         # Включить/выключить запись
RECORDINGS_DIR=./recordings   # Директория для записей

# Производительность
CAPTURE_INTERVAL=5000         # Интервал захвата (мс), больше = меньше нагрузка
IMAGE_QUALITY=0.8             # Качество 0.0-1.0, меньше = меньше размер
IMAGE_WIDTH=1080              # Ширина изображения (px)
INACTIVITY_TIMEOUT=15000      # Время до предупреждения о неактивности (мс)

# Конвертация видео
VIDEO_FPS=2                   # Кадров в секунду в итоговом видео
VIDEO_FORMAT=mp4              # Формат видео
```

### Рекомендации по настройке производительности

**Для большого количества участников (20+):**
```bash
CAPTURE_INTERVAL=10000    # 10 секунд
IMAGE_QUALITY=0.6
IMAGE_WIDTH=720
```

**Для среднего количества (10-20):**
```bash
CAPTURE_INTERVAL=5000     # 5 секунд (по умолчанию)
IMAGE_QUALITY=0.8
IMAGE_WIDTH=1080
```

**Для высокого качества (до 10 участников):**
```bash
CAPTURE_INTERVAL=3000     # 3 секунды
IMAGE_QUALITY=0.9
IMAGE_WIDTH=1920
```

## 🎯 Использование

### Для участников экзамена

1. Откройте `http://IP_СЕРВЕРА:3000`
2. Введите фамилию и имя
3. Нажмите "Делиться экраном"
4. Выберите экран/окно для демонстрации
5. Не закрывайте вкладку до конца экзамена

### Для преподавателя

**Мониторинг в реальном времени:**
```
http://IP_СЕРВЕРА:3000/screens?password=inga
```

- Просмотр всех экранов сеткой
- Клик на экран для полноэкранного просмотра
- Список подключенных участников
- Журнал подключений/отключений
- Предупреждения о неактивных участниках

**Управление записями:**
```
http://IP_СЕРВЕРА:3000/manage?password=inga
```

- Просмотр всех сессий записи
- Статистика по каждому участнику
- Конвертация скриншотов в видео
- Скачивание готовых видео
- Удаление записей

## 🎬 Конвертация в видео

### Через веб-интерфейс

1. Откройте `/manage?password=inga`
2. Нажмите "🎬 В видео" для участника
3. Или "🎬 Конвертировать всё" для всей сессии

### Через командную строку

**Конвертировать одного участника:**
```bash
node convert_to_video.js recordings/session_2025-01-15/Иванов_abc123
```

**Конвертировать всю сессию:**
```bash
node convert_to_video.js --session recordings/session_2025-01-15
```

**С настройкой параметров:**
```bash
VIDEO_FPS=5 VIDEO_FORMAT=mp4 node convert_to_video.js <путь>
```

## 🐳 Docker

### Запуск

```bash
# Сборка и запуск
docker-compose up -d

# Просмотр логов
docker-compose logs -f

# Остановка
docker-compose down
```

### Доступ к записям

Записи сохраняются в папку `./recordings` на хосте, даже после остановки контейнера.

### Обновление

```bash
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

## 🔒 Безопасность

- Измените пароль в `.env` перед использованием
- Используйте HTTPS в продакшене (добавьте nginx с SSL)
- Ограничьте доступ к серверу на уровне сети
- Регулярно удаляйте старые записи

## 🛠️ Требования

- **Node.js** 18+ (для локального запуска)
- **ffmpeg** (для конвертации видео)
- **Docker** и **Docker Compose** (для запуска в контейнере)

## 🤝 Поддержка

При возникновении проблем проверьте:

1. Установлен ли ffmpeg: `ffmpeg -version`
2. Доступны ли порты: `netstat -an | grep 3000`
3. Правильно ли указан пароль в URL
4. Достаточно ли места на диске для записей
5. Хватает ли оперативной памяти (рекомендуется 2GB+)

## 📈 Оценка требований

**Ориентировочное потребление:**

- 1 участник: ~50 KB/скриншот × 12 скриншотов/минуту = ~600 KB/мин
- 20 участников: ~12 MB/мин = ~720 MB/час
- Итоговое видео: ~50-100 MB/час на участника (зависит от FPS и качества)

**Рекомендуемое железо:**

- CPU: 2+ ядра
- RAM: 2-4 GB
- Диск: 10+ GB свободного места
- Сеть: 10+ Mbps для 20 участников
